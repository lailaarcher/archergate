What's inside

Everything between your binary and a cracked copy.

Machine binding

SHA-256 fingerprint from CPU identity and OS install ID. License stays on the machine it was activated on.

Offline validation

Validate once, work offline for 30 days. HMAC-signed cache. Touring musicians, remote studios, air-gapped labs.

14-day trials

Built into the SDK. No server, no signup, no tracking. Users try it, then decide.

Tamper detection

HMAC-signed cache. Validation receipts. Heartbeat counter. Three independent verification paths.

Rust, C, C++

Native Rust SDK. C FFI with RAII wrapper. REST API for any other language. Static library, no runtime deps.

Self-hosted server

Reference server included. Axum + SQLite. Run it yourself. Or use Archergate's hosted option when it ships.

Integration

Three lines. Thirty minutes.

Rust
use archergate_license::LicenseClient; let client = LicenseClient::new( "your-api-key", "com.you.app", ); client.validate("XXXX-XXXX")?;

REST API (any language)
POST /validate { "license_key": "XXXX", "machine_fingerprint": "...", "plugin_id": "com.you.app" } → { "valid": true }

Common questions

Is the SDK free?

Yes. MIT licensed, forever. We may charge for managed hosting later. The SDK and self-hosted server stay free.

What languages are supported?

Rust natively. C and C++ via FFI. Any language via REST API (Python, C#, Go, JS). Static library links into your binary.

Can I run my own server?

Yes. Reference server ships with the SDK. Rust + SQLite. Deploy it on any $5 VPS. Modify it however you want.

What data do you collect?

Self-hosted: nothing. Archergate cloud (future): license key, machine fingerprint, activation date. No analytics, no tracking.

What about offline users?

One online validation, then 30 days offline. HMAC-signed local cache. No internet needed during that window.

What if Archergate disappears?

The SDK is open source. Self-host, fork, or replace the server. Your licensing depends on your code, not ours.
