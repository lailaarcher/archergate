Copy protection that ships with your binary.

Archergate is a licensing library. Drop it into your C, C++, or Rust project. It locks each license key to the machine it was activated on. Works offline for 30 days. Self-hostable. MIT licensed.

Download Windows MSVC x64 .tar.gz macOS Intel + Apple Silicon .tar.gz Linux x86_64 .tar.gz
Rust: cargo add archergate-license

What this does.

You sell software. Plugins, desktop apps, game tools, creative addons. Right now, anyone with your binary can use it. Archergate ties each license key to the hardware it was activated on. If someone copies the binary to another machine, it won't run without a valid key for that machine.

The SDK handles machine fingerprinting, key validation, offline caching, trial periods, and tamper detection. You handle selling and distributing keys however you already do: Gumroad, Stripe, your own store, anything.

Questions.

What does this replace?

Custom license code you would write yourself, or services like Keygen, iLok, or Gumroad's license API. This is a static library, not a platform.

Where does the server run?

Wherever you want. The reference server is Rust + SQLite. A $5 VPS, a Docker container, your existing infrastructure. Modify it however you need.

What data touches your servers?

Nothing, if you self-host. The SDK talks to your server. License key, machine fingerprint, activation date. No analytics, no telemetry.

What if Archergate disappears?

The SDK is MIT licensed. Fork it, vendor it, rewrite the server. Your product depends on your copy of the code, not ours.
